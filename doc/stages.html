This document describes the implementation (or not) of each stage. Generally, a stage falls into one of four categories:

  • Implemented - 100% faithful to the original
  • Implemented - Some minor differences discrepancies (usually due to platform differences)
  • Not implemented - I haven't had time to implement yet
  • Not implemented - No plans to implement (usually due to platform differences)

Where there are no implementation notes below, the funtionality is an exact replication of the original CMS behaviour
